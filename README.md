# Go-Master-Slave-Project

Project Title: Building a Distributed Database Using Go
Objective:
Design and implement a basic distributed database system using the Go 
programming language. This project will introduce core concepts of 
distributed systems, including data replication and fault tolerance.
Key Requirements:
1. Language: Go (Golang)
2. Architecture:
o Minimum of 3 nodes communicating over the network.
o Each node should be able to store data independently.
o Nodes must communicate using TCP or HTTP….
3. Core Features:
o Master Create new DB (or use existing) then create tables on it
dynamically each table with its attributes
o All nodes(master and slaves ) can do some queries like search,
update ,delete record or table ,select , insert and master only 
can drop the DB
o Replication (replicate data across nodes)
